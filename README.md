## 本サービスについて

本サービスは、**エンジニアの方が普段の技術学習や開発業務に取り組む際に、チャット形式で気軽にメモをとり、その内容を 今日学んだこと（TIL）として GitHub リポジトリに記録できるタスク管理×作業メモアプリです。**

以下の一連の流れをアプリ内で完結していることが本サービスの特徴です。

### Step 1. タスクページを作る 🗒️

取り組む内容に応じてタスクページを作り、ページで達成する項目を ToDo リストに設定します。

https://github.com/user-attachments/assets/adb68062-fb16-4ef2-92e2-97eb40dd7677

### Step 2. タスクページで気づいたことをメモする 📝

タスクページで作業中に気づいたことや調べたことのメモ、またコードスニペットや参考にした記事の投稿を行うことができます。

https://github.com/user-attachments/assets/bee08207-d372-4956-8875-28e7bd9352dc

### Step 3. メモをもとに 今日学んだこと（TIL）を書き、GitHub リポジトリに記録する 🍀

作業を完了すると、メモを参考に 今日学んだこと（TIL） を書き、GitHub リポジトリに学習成果を記録できます。

https://github.com/user-attachments/assets/388c2e70-9d69-4719-9a1f-eb9d66261c0b

## 本サービスの開発の背景

私は普段の開発や学習を進める際、GitHub Issue でタスク管理を行いながら、作業中の気づきや調べた内容は Notion や Mattermost など複数の別々のツールにメモしていました。
しかし、この方法では「このタスクで何を行い、何に気づいたか」が複数のツールに分散されてしまい、後から振り返る際に複数のツールを探し回る必要があり非効率でした。また、作業中に得た知見も整理されないまま埋もれてしまいがちで、せっかくの学びが十分に活かされないという課題がありました。

こうした課題を解決するには、**タスクと作業メモを同じ場所で管理できる仕組みが必要**だと考えました。

加えて、エンジニアの間で広まっている TIL（Today I Learned）文化にも共感しており、日々の作業メモをそのまま学習記録として残せれば、メモをとるモチベーション自体も上がると気づきました。つまり、**メモをとることと TIL を書くことには強い親和性がある**と考えたのです。

そこで、GitHub Issue のタスク管理 UI と Claude のチャット形式の UI を参考にしながら、**タスク管理・作業メモ・学習記録の 3 つの機能を一つに統合したアプリ「めもTIL」** を開発しました。



## 主な機能

### A. タスクページ内の操作

1. [タスクページの作成・編集・削除](#a-1)
2. [タスクページ内の ToDo の作成・編集](#a-2)
3. [タスクページ内のメモの投稿・削除](#a-3)
4. [タスクページのステータス変更](#a-4)

### B. GitHub アカウントとの連携によるリポジトリ, md ファイル作成

1. [GitHub 認証, 連携](#b-1)
2. [タスクページ内の GitHub 上のリポジトリ, md ファイル作成](#b-2)

<a id="a-1"></a>

### A.1. タスクページの作成・編集・削除

- タスクページ作成・編集時に、タイトル・概要の入力とタグの設定ができます。
- タグはオートコンプリートで候補を表示し、既存タグの追加や新規のタグ名の入力ができます（1 タスクあたり最大 5 件まで）。
- 「タスクを削除する」ボタンをクリックすると、タスクページを削除できます。

<a id="a-2"></a>

### A.2. タスクページ内の ToDo の作成・表示・編集・削除

- ToDo フォームの「追加」ボタンをクリックすると、ToDo フィールドを追加できます。
- ToDo フィールドは最大 5 個まで追加可能です。
- 各 ToDo フィールドの「×」ボタンをクリックすると、ToDo フィールドを削除できます。
- ToDo リストの「✒️」ボタンをクリックすると、ToDo フォームに切り替わります。

<a id="a-3"></a>

### A.3. タスクページ内のメモの投稿・表示・削除

- 「メモ」タブから"テキストメモ"と"コードスニペット"の投稿を行えます。
- 「参照 URL」タブから参照した"ドキュメント"の投稿を行えます。
- "コードスニペット"はシンタックスハイライト表示されます。
- "ドキュメント"は OGP カード形式で表示されます。
- 各ポストの下部にある「🪣」ボタンをクリックすると、ポストを削除できます。

<a id="a-4"></a>

### A.4. タスクページのステータス変更

- タスクには「Doing」と「Done」のステータスがあり、ボタンをクリックすると切り替わります。

<a id="b-1"></a>

### B.1. GitHub 認証, 連携

- ログイン画面から GitHub アカウントによる認証が可能です。
- また、メールアドレスでログインしている方は、ステータスが「Done」の状態のタスクページの右サイドバーから GitHub アカウントとの連携が可能です。

<a id="b-2"></a>

### B.2. タスクページ内の GitHub 上のリポジトリ, md ファイル作成

- GitHub アカウントと連携しているユーザーは、ステータスが「Done」の状態のタスクページの右サイドバーから、 GitHub 上のリポジトリを作成できます。
- また、GitHub 上のリポジトリを選択し、リポジトリ下に md ファイルの作成もできます。

<a id="c-1"></a>

### C.1. 確認メールによる認証

- サインアップ後、登録したメールアドレス宛に確認メールが送信されます。
- 確認メール内のリンクをクリックすると確認が完了し、そのメールアドレスでログインできます。
- 未確認のメールアドレスでログインしようとすると、アラートメッセージが表示され、確認メールの再送信画面へのリンクが表示されます。

<a id="c-2"></a>

### C.2. リクエストメールによるパスワードリセット

- ログイン画面からパスワードリセットリクエスト画面へアクセスできます。
- 登録済み、かつ、確認済みのメールアドレス宛にリクエストメールを送信できます。
- リクエストメール内のリンクをクリックするとパスワードリセット画面へアクセスし、パスワードの変更ができます。

<a id="c-3"></a>

### C.3. ゲストユーザーの自動生成

- ログイン画面からゲストユーザーを自動生成し、そのアカウントでログインできます。
- ゲストユーザーが利用可能な機能は制限し、ゲストユーザーが GitHub 連携、および、GitHub 上のリポジトリ, md ファイルの作成を行うことはできません。

<a id="c-4"></a>

### C.4. ユーザーの編集・削除

- ヘッダー右のアイコンからアカウント設定画面へアクセスできます。
- ユーザー名とメールアドレスの変更ができます。（メールアドレスの変更方法は後述します）
- 「アカウントを削除」ボタンをクリックすると、ユーザーを削除できます。

<a id="c-5"></a>

### C.5. 確認メールによるメールアドレスの変更

- アカウント設定画面から新しいメールアドレスを入力すると、そのメールアドレス宛に確認メールが送信されます。
- 確認メール内のリンクをクリックすると確認が完了し、メールアドレスが変更されます。

<a id="d-1"></a>

### D.1. タスクページのキーワード検索（オートコンプリート）

- タスクページの「タイトル」に対して、キーワード検索できます。オートコンプリート機能により、文字を入力するとリアルタイムで検索結果を表示します。

<a id="d-2"></a>

### D.2. タスクページのフィルタリング

- タスクページの「ステータス」でフィルタリングできます。

<a id="d-3"></a>

### D.3. タスクページのページネーション

- ページネーションにより、1 ページあたり 24 件まで表示する。
