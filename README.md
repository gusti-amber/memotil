## 本サービスについて

本サービスは、**プログラミング学習やジュニアエンジニアの方が普段のプログラミング学習や開発作業に取り組む際に、チャット形式で気軽にメモをとり、その内容を TIL（今日学んだこと）として GitHub リポジトリに記録できる学習・作業メモアプリです。**

以下の一連の流れをアプリ内で完結していることが本サービスの特徴です。

### Step 1. タスクページを作る 🗒️

取り組む内容に応じてタスクページを作り、ページで達成する内容を ToDo リストに書きます。

### Step 2. タスクページで気づいたことをメモする 📝

ページ内で気づいたことや調べたことのメモ、またコードスニペットや参考にした記事の投稿を行うことができます。

### Step 3. メモをもとに TIL（今日学んだこと）を書き、GitHub リポジトリに記録する 🍀

学習・作業を完了すると、メモをもとに TIL を書き、GitHub リポジトリに学習成果を記録できます。

## 本サービスの開発の背景

私は本サービスを開発する以前、Web 技術の学習やアプリ開発に取り組む中で、気づいたことや調べた内容を [Notion](https://www.notion.com/ja) や [Arc ブラウザの Easel 機能(キャンバス機能)](https://resources.arc.net/hc/en-us/articles/19231142050071-Easels-Capture-Create)を使ってメモしていました。しかし、これらのツールにはそれぞれ使いづらさを感じていました。

Notion は ブログ形式で記述できる反面、文章としてまとめることを意識してしまい、学習や作業中に生じた気づきや調べた内容を**気軽にメモする用途としては向いていない**と感じていました。  
一方、Arc の Easel 機能は、キャンバスにテキストやドキュメントの URL を貼り付けることで、直感的にメモを残せる面は魅力的でしたが、**メモ同士の関係性や作業の流れを把握しづらい**という課題がありました。

こうした課題を解決するために、本サービスでは**タスクページ内でテキストメモ、参照したドキュメント、コードスニペットをチャット形式で投稿できる仕組み**を採用しました。この仕組みにより、気軽にメモをとり、学習・作業の流れを時系列で視覚的に把握することができます。

また、メモの表示方式にも工夫を加えており、ドキュメントは OGP カード形式で表示し、コードスニペットはシンタックスハイライトを適用することで、情報を一目で整理できるようにしています。

さらに、**メモをとることと TIL を書くことの親和性に着目**し、本サービスでは GitHub アカウントと連携することで、**GitHub リポジトリ上に TIL を記録できる機能**を提供しています。

学習・作業中のメモから GitHub リポジトリへの TIL の記録までを一貫して行えることが本サービスの大きな特徴です。

<a id="a-1"></a>

### A.1. タスクページの作成・編集・削除

- タスクページ作成・編集時に「タイトル」の入力と「タグ」の選択を行えます。
- 「タスクを削除する」ボタンをクリックすると、タスクページを削除できます。

<a id="a-2"></a>

### A.2. タスクページ内の ToDo の作成・表示・編集・削除

- ToDo フォームの「追加」ボタンをクリックすると、ToDo フィールドを追加できます。
- ToDo フィールドは最大 5 個まで追加可能です。
- 各 ToDo フィールドの「×」ボタンをクリックすると、ToDo フィールドを削除できます。
- ToDo リストの「✒️」ボタンをクリックすると、ToDo フォームに切り替わります。

<a id="a-3"></a>

### A.3. タスクページ内のメモの投稿・表示・削除

- 「メモ」タブから"テキストメモ"と"コードスニペット"の投稿を行えます。
- 「参照 URL」タブから参照した"ドキュメント"の投稿を行えます。
- "コードスニペット"はシンタックスハイライト表示されます。
- "ドキュメント"は OGP カード形式で表示されます。
- 各ポストの下部にある「🪣」ボタンをクリックすると、ポストを削除できます。

<a id="a-4"></a>

### A.4. タスクページのステータス変更

- タスクには 3 つのステータス「Todo」, 「Doing」, 「Done」が存在し、ボタンをクリックすると、ステータスが変更されます。

<a id="b-1"></a>

### B.1. GitHub 認証, 連携

- ログイン画面から GitHub アカウントによる認証が可能です。
- また、メールアドレスでログインしている方は、ステータスが「Done」の状態のタスクページの右サイドバーから GitHub アカウントとの連携が可能です。

<a id="b-2"></a>

### B.2. タスクページ内の GitHub 上のリポジトリ, md ファイル作成

- GitHub アカウントと連携しているユーザーは、ステータスが「Done」の状態のタスクページの右サイドバーから、 GitHub 上のリポジトリを作成できます。
- また、GitHub 上のリポジトリを選択し、リポジトリ下に md ファイルの作成もできます。

<a id="c-1"></a>

### C.1. 確認メールによる認証

- サインアップ後、登録したメールアドレス宛に確認メールが送信されます。
- 確認メール内のリンクをクリックすると確認が完了し、そのメールアドレスでログインできます。
- 未確認のメールアドレスでログインしようとすると、アラートメッセージが表示され、確認メールの再送信画面へのリンクが表示されます。

<a id="c-2"></a>

### C.2. リクエストメールによるパスワードリセット

- ログイン画面からパスワードリセットリクエスト画面へアクセスできます。
- 登録済み、かつ、確認済みのメールアドレス宛にリクエストメールを送信できます。
- リクエストメール内のリンクをクリックするとパスワードリセット画面へアクセスし、パスワードの変更ができます。

<a id="c-3"></a>

### C.3. ゲストユーザーの自動生成

- ログイン画面からゲストユーザーを自動生成し、そのアカウントでログインできます。
- ゲストユーザーが利用可能な機能は制限し、ゲストユーザーが GitHub 連携、および、GitHub 上のリポジトリ, md ファイルの作成を行うことはできません。

<a id="c-4"></a>

### C.4. ユーザーの編集・削除

- ヘッダー右のアイコンからアカウント設定画面へアクセスできます。
- ユーザー名とメールアドレスの変更ができます。（メールアドレスの変更方法は後述します）
- 「アカウントを削除」ボタンをクリックすると、ユーザーを削除できます。

<a id="c-5"></a>

### C.5. 確認メールによるメールアドレスの変更

- アカウント設定画面から新しいメールアドレスを入力すると、そのメールアドレス宛に確認メールが送信されます。
- 確認メール内のリンクをクリックすると確認が完了し、メールアドレスが変更されます。

<a id="d-1"></a>

### D.1. タスクページのキーワード検索（オートコンプリート）

- タスクページの「タイトル」に対して、キーワード検索できます。オートコンプリート機能により、文字を入力するとリアルタイムで検索結果を表示します。

<a id="d-2"></a>

### D.2. タスクページのフィルタリング

- タスクページの「ステータス」と「タグ」でフィルタリングできます。

<a id="d-3"></a>

### D.3. タスクページのページネーション

- ページネーションにより、1 ページあたり 24 件まで表示する。
